Python的魔法方法（下）

本文是对B站UP主“码农高天”的《Python的魔法方法》系列视频的总结和整理，搭配视频食用本文效果更佳。

魔法方法：Python提供的让用户客制化一个类的方式，定义在类内的一些特殊方法，这些方法的名称前后会有两个下划线。

与类的建立有关的方法

__ init_subclass__

这个魔法方法需要定义在基类里，当以这个类为基类，定义一个派生类的时候，这个方法就会被调用。运行下面的代码就会打印出类A。其中cls参数就是定义的派生类。

class Base:
    def __init_subclass__(cls):
        print(cls)
        
class A(Base):
    pass

# 执行：
<class '__main__.A'>

可以在该方法里定义一个空字典，如下所示。同时，该方法也可以传入参数。

class Base:
    def __init_subclass__(cls, name):
        cls.x = {}
        cls.name = name
        
class A(Base, name="Jack"):
    pass

print(A.x) # {}
print(A.name) # Jack

__ set_name__

该方法更多是用在描述器里。当在某个类的定义内，去实例化该类的对象时，就会调用这个方法。执行下面的代码，发现owner参数是类A，即表示在哪个类内去实例化对象；而name表示实例化对象时所赋值的变量名。

class D:
    def __set_name__(self, owner, name):
        print(owner, name)
class A:
    x = D()
# 执行：
<class '__main__.A'> x

__ class_getitem__ 和__ mro_entries__

__getitem__：是当类实例化的对象使用[]进行取值时调用的方法。

__class_getitem__：是当类使用[]进行取值时调用的方法。可以用来对type hint做增强。

class A:
    def __class_getitem__(cls, item):
        print(item)
        return "abc"
print(A[0])
# 0
# abc

如下的代码，在类型提示中list[int]表示一个由int组成的列表，可以将其赋给一个变量作为类型提示。

int_arr_type = list[int]
list1: int_arr_type = []
list2: int_arr_type = []

__mro_entries__：某个类在做继承的时候，大多数是class B(A)的形式，如果直接写做class B(A())的就会报错，报错是因为建立类B的时候寻找自己的基类出现了问题，因此__mro_entries__方法就是在建立类的过程中帮助寻找基类的，该方法会返回一个tuple，表示去哪里找基类。执行下面的代码则不会再报错并能运行了。

class A:
    def __mro_entries__(self, bases):
        print(base)
        return ()
    
class B(A()):
    pass

# 执行：
# (<__main__.A object at 0x7f87ab86a400>,)

从上面代码可以看到，__mro_entries__返回的是一个空tuple，因此在建立类B的过程中，类A的实例化对象提供不了类B的基类，如下面的代码打印结果是False。

class A:
    def __mro_entries__(self, bases):
        print(base)
        return ()
    
class B(A()):
    pass

print(issubclass(B, A)) # False

那再修改一下代码看看，说明B是A的子类，B成功找到了自己的基类。

class A:
    def __mro_entries__(self, bases):
        print(base)
        return (A,)
    
class B(A()):
    pass

print(issubclass(B, A)) # True

运算方法

常用的运算方法

在Python中，如果直接对一个对象做数学运算，Python会报错。如下面的代码中的两个向量相加。

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"
    
v1 = Vector(0,1)
v2 = Vector(2,3)
print(v1+v2) # 报错

这个时候可以在类Vector中定义__add__方法，帮助自定义的向量对象做加法操作。

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"
    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)
    
v1 = Vector(0,1)
v2 = Vector(2,3)
print(v1+v2) 
# Vector(2, 4)

在Python中可以定义的与数值运算相关的魔法方法如下，这也是对Python中的运算符号(+、-、*、/等)进行重写：

加法__add__、减法__sub__、乘法__mul__、矩阵乘法__matmul__、除法__truediv__、整除__floordiv__、取余__mod__、既拿到商也拿到余数__divmod__、乘方__pow__。

左移lshift、右移rshift。

逻辑计算与__and__、逻辑计算异或__xor__、逻辑计算或__or__，注意这里不是对Python的关键字and和or的重写，是对符号&和|的重写。

这里的魔法方法除了乘方比较特殊外，其余的魔法方法定义形式基本相同。

class A:
    def __xxx__(self, other):
        # self和other做定义的运算

乘方的魔法方法的定义如下，相比于其他方法多了一个参数mod，该参数是指在做完乘方运算后可以再进行取余操作。

class A:
    def __pow__(self, other, mod=None):
        pass
        # pow(v1, v2, mod) => (v1**v2)%mod

这里有几个注意的地方：首先，self和other并不一定是相同的类型，比如self是一个Vector对象，而other是一个int整数。可以在这个方法里实现数乘和点乘的功能。

class Vector:
    def __mul__(self, other):
        if isinstance(other, int)
            return Vector(self.x * other, self.y * other)
       	elif isinstance(other, Vector):
            return Vector(self.x * other.x, self.y * other.y)

另外，使用魔法方法只是借用这个符号，并不一定要符合原来的定义，比如在C++ 中将左移运算符重新定义为cout.print，因此在使用<<符号时本质是在执行打印功能。

最后看一个例子，我们在下面的代码中执行print(v1*2)时本质是v1.__mul__(2)，可以正确得到结果，那么如果是执行print(2*v1)则会报错，对于这种情况应该怎么办呢？

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"    
    def __mul__(self, other):
        if isinstance(other, int)
            return Vector(self.x * other, self.y * other)
       	elif isinstance(other, Vector):
            return Vector(self.x * other.x, self.y * other.y)
v1 = Vector(0,1)
print(v1*2) # 等价于 v1.__mul__(2)
print(2*v1) # 报错

此时，可以定义方法__rmul__来解决：如果操作符左侧的对象没有定义运算操作怎么完成时，就会尝试去找操作符右侧的数据对象的r版本的运算方法，也即执行other*self时，other不知道怎么做乘法时，就会调用self的__rmul__方法，然后把other作为参数传入，方法内部的实现与__mul__方法完全相同。

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"    
    def __mul__(self, other):
        if isinstance(other, int):
            return Vector(self.x * other, self.y * other)
       	elif isinstance(other, Vector):
            return Vector(self.x * other.x, self.y * other.y)
    def __rmul__(self, other):
        if isinstance(other, int):
            return Vector(self.x * other, self.y * other)
       	elif isinstance(other, Vector):
            return Vector(self.x * other.x, self.y * other.y)        
v1 = Vector(0,1)
print(v1*2) # Vector(0, 2)
print(2*v1) # Vector(0, 2)

这里提到的所有和数值计算有关的魔法方法，都有对应的r版本，也即在方法名前加一个字母“r”。比如加法__add__和__radd__。

除了魔法方法的r 版本外，还有一个i版本，这是指原地（in-place）进行运算操作，修改self自身：相比于上述方法，不是返回一个新的内存数据，而是修改自身的内存数据。对应的运算符号是原来的操作符后面加一个等号，比如加法，v1 += v2就会调用__iadd__方法：v1.__iadd__(v2)。

class A:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"    
    def __iadd__(self, other):
        # v1 += v2
        self.x += other.x
        self.y += other.y
        return self

在所有使用运算符号的操作中，都有对应的i版本，比如+=、-=、*=、/=等操作，对应的i版本方法在方法名前加一个字母“i”。

其他的一元运算

__pos__：对应的操作是在数据对象之前加一个正号

__neg__：对应的操作是在数据对象之前加一个负号

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"    
    def __pos__(self):
        print("Pos")
        return Vector(self.x, self.y)
    def __neg__(self):
        return Vector(-self.x, -self.y)
    
v1 = Vector(1,2)
print(+v1)
# Pos
# Vector(1,2)

__abs__：求绝对值操作，对应的运算符是abs，在Python中使用abs(xxx)会调用__abs__方法。

__invert__：取反操作，对应的运算符号是~，一般用作位运算。

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"    
    def __abs__(self):
        return Vector(abs(self.x), abs(self.y))
    def __neg__(self):# 这里自定义顺序取反
        return Vector(self.y, self.x)

__int__：使用Python的内置函数int()时，调用该方法

__float__：使用Python的内置函数float()时，调用该方法

__complex__：使用Python的内置函数complex()时，调用该方法

这三个魔法方法规定，方法的返回值必须是它们对应的数据结构。假如想定义int()操作是对Vector里的x和y都进行int操作，如下代码定义，则执行这段代码会报错。

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"    
    def __int__(self):
        return Vector(int(self.x), int(self.y))

__index__：表示当把数据对象当作索引使用时，它产生的行为功能是什么。

当定义了__index__方法，且__int__、__complex__和__float__没有被定义时，使用Python的内置方法int()、float()和complex()时会默认执行__index__方法。

class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    def __repr__(self):
        return f"Vector({self.x}, {self.y})"    
    def __index__(self):
        return int(self.x)
    
v = Vector(1.5, 2.5)
lst = [0,1,2,3]
print(lst[v]) # 1

取整操作

__round__：对应Python的内置函数round，四舍五入，取最近的整数

__trunc__：math库内的trunc函数，小数点后面不要，即向0取整

__floor__：math库内的floor函数，向下取整，即向负无穷取整

__ceil__：math库内的ceil函数，向上取整，即向正无穷取整

模拟行为的方法

__ call__

__call__：该方法可以以函数调用的形式使用对象。

如下面代码所示，定义了一个乘法器，乘法倍数是3，该类实例化得到对象o。因为定义了__call__方法，则对象o可以以函数形式被使用。

class Multiplier:
    def __init__(self, mul):
        self.mul = mul
    def __call__(self, arg):
        return self.mul*arg
    
o = Multiplier(3)
print(o(4)) # 12

__ len__

__len__：描述该对象（容器）的长度。

如下代码所示，自定义一个列表容器，这里使用Python内置的list的__len__实现，返回self.data的长度。__len__会被Python内置方法len()调用

class MyList:
    def __init__(self, data):
        self.data = data
    def __len__(self):
        return len(self.data)
    
x = MyList([1,2,3])
print(len(x)) # 3

如果自定义的类内没有定义__bool__方法，而且又把该类的对象当作条件去做判断时，该对象就会尝试调用__len__方法，在__len__被定义的情况下，如果返回长度为0则认为False，否则认为True。

class MyList:
    def __init__(self, data):
        self.data = data
    def __len__(self):
        return len(self.data)
    
x = MyList([1,2,3])
if x:
    print("Hello")
# Hello

该特性和所有Python内置的容器是一样的。比如下面的代码：

x = list()
d = dict()
if x and d:
    ......

__ getitem__ 、 __ setitem __ 和 __ delitem__

这三个方法都是在描述对象使用[]时具体的行为功能。

__getitem__：对象使用[]去读取数据时会调用该方法。

class MyList:
    def __init__(self, data):
        self.data = data
    def __getitem__(self, index):
        return self.data[index]
    
x = MyList([1,2,3])
print(x[1]) # 2

__setitem__：对象使用[]去赋值数据时会调用该方法。

class MyList:
    def __init__(self, data):
        self.data = data
    def __getitem__(self, index):
        return self.data[index]        
    def __setitem__(self, index, value):
        self.data[index] = value
    
x = MyList([1,2,3])
x[1] = 10
print(x[1]) # 10

__delitem__：对象使用[]去删除数据时会调用该方法：即被关键字del触发。

class MyList:
    def __init__(self, data):
        self.data = data
    def __getitem__(self, index):
        return self.data[index]        
    def __delitem__(self, index):
        self.data = self.data[0:index] + self.data[index+1:]
    
x = MyList([1,2,3])
print(x[1]) # 2
del x[1]
print(x[1]) # 3

__ reversed__

__reversed__：将对象（容器）里的数据翻转，会被Python内置的reversed()函数调用。

class MyList:
    def __init__(self, data):
        self.data = data
    def __reversed__(self):
        return MyList(self.data[::-1])    
x = MyList([1,2,3])
print(reversed(x).data)
# [3,2,1]

__ contains__

__contains__：描述对象在使用in操作时的功能行为，即对象在使用in会调用该方法。

如下代码所示，自定义一个列表容器，这里使用Python内置的list的__contains__实现：item in self.data。

class MyList:
    def __init__(self, data):
        self.data = data
    def __contains__(self, item):
        return item in self.data   
x = MyList([1,2,3])
print(2 in x) # True
print(5 in x) # False

__ iter__

__iter__：返回该对象（容器）的一个迭代器，会被Python内置的iter()函数调用。

如下代码所示，自定义一个列表容器，这里使用Python内置的list的__iter__实现：iter(self.data)。注意：在使用for循环时，Python会隐性调用iter()函数，所以for i in x其实是for i in iter(x)。

class MyList:
    def __init__(self, data):
        self.data = data
    def __iter__(self):
        return iter(self.data)  
x = MyList([1,2,3])
for i in x:
    print(i)

上下文__ enter__ 和__ exit__

平时的代码with open(xxxx) as f中，with后面的部分就是上下文，要想定义一个上下文，需要定义两个魔法方法：__enter__和__exit__。

__enter__：与大于运算__gt__方法类似，greater than or equal to

__exit__：与小于运算__lt__方法类似，less than or equal to

如下的代码，自定义一个计时器，执行代码with Timer():，这时会显示花费的时间。

import time

class Timer:
    def __enter__(self):
        self.start = time.time()
    def __exit__(self, exc_type, exc_value, traceback):
        print(f"Time: {time.time() - self.start}")
with Timer():
    x = 1000*10
# 执行
# Time: 1.4342545363673e-06

with Timer() as t:
    x = 1000*10

一般的代码是with open() as f，如同下面的代码，执行代码with Timer() as t:，这个as t就是将__enter__的返回值保存在t里，一般常见的操作是将对象本身self返回。

class Timer:
    def __enter__(self):
        self.start = time.time()
        return self
    def __exit__(self, exc_type, exc_value, traceback):
        print(f"Time: {time.time() - self.start}")

with Timer() as t:
    print(t.start)
    x = 1000*10
# 执行
# 1231423426.45353
# Time: 1.4342545363673e-05

__exit__方法里的几个参数，都是和异常相关的，比如下面的代码，值的注意的是，程序依然得出了花费的时间。即即使代码抛出了异常，__exit__方法内的代码依然被执行了，比如在文件操作中，常常使用with open() as f的代码，而不是f = open()，就可以防止忘记关闭文件，同时就算程序报错，也可以成功把文件关闭

class Timer:
    def __enter__(self):
        self.start = time.time()
        return self
    def __exit__(self, exc_type, exc_value, traceback):
        print(exc_type, exc_value, traceback)
        print(f"Time: {time.time() - self.start}")

with Timer() as t:
    x = 1000 / 0
# 执行
# <class 'ZeroDivisionError'> division by zero <traceback object at 0x7f6961387800>
# Time:6.103745625e-05
# Traceback (most recent call last):
#   File "<stdin>", line 2, in <module>
# ZeroDivisionError: division by zero
