Python的魔法方法（上）

本文是对B站UP主“码农高天”的《Python的魔法方法》系列视频的总结和整理，搭配视频食用本文效果更佳。

魔法方法：Python提供的让用户客制化一个类的方式，定义在类内的一些特殊方法，这些方法的名称前后会有两个下划线。

基础方法

__ init__ 和__ new__

这两个魔法方法能够改变一个类实例化对象时的行为。

__new__：是描述从一个class建立一个object的过程

__init__：是描述当存在一个object后，对其进行初始化的过程

class A:
    def __new__(cls, x):
        print("__new__")
        return super().__new__(cls)
    
    def __init__(self, x):
        self.x = x
        print("__init__")
        
o = A(xxx)
# 等价于
# 1. obj = __new__(A, xxx)
# 2. __init__(obj, xxx)

如果在建立object的过程中传入了若干参数，则这些参数既会被传入到__new__中，也会被传入到__ini__中。__new__因为是建立object，所以是有返回值的。

__ del__

__del__可以粗略的认为是一个析构函数，是描述一个object被释放的过程。但是由于Python的对象释放过程较为复杂，因此该方法不可控。

class A:
    def __del__(self):
        print("__del__")
o = A()

注意：这里的__del__和关键字del没有关系，del o并不一定会触发__del__方法，只是让对象的引用-1。

__ repr__ , __ str__ 和__ format__

__repr__和__str__这两个方法的功能是相似的，都是返回一个object的字符串表示，这两个方法主要是语义的不同。

__repr：representation，一般要有更详细的信息，可以通过repr()内置函数调用该方法。

__str__：注重可读性，可以通过str()内置函数调用该方法，print内部就是使用了str()方法。

__format__：使用某种格式打印object时，会调用该方法。

print("{}".format(15)) # 10进制
print("{:b}".format(15)) # 2进制
print("{:x}".format(15)) # 16进制

print(f"{15}") # 10进制
print(f"{15:b}") # 2进制
print(f"{15:x}") # 16进制

class A:
    def __format__(self, spec):
        if spec == "x":
            return "0xA"
        return "<A>"

print(f"{A()}") # <A>
print(f"{A():x}") # 0xA

__ bytes__

__bytes__：通过class去建立bytes时的行为，客制化object的bytes表示。通过内置方法bytes()调用该魔法方法。

class A:
    def __bytes__(self):
        print("__bytes__")
        return bytes([0,1])
    
print(bytes(A())) # b'\x00\x01'

比较方法

等于 __ eq__ 和不等于__ ne__

__eq__：描述两个object进行比较时的过程，判断两个object是否相等。

__ne__：描述两个object进行比较时的过程，判断两个object是否不相等。

上述两个方法，等于和不等于，是有默认实现的，如果没有自定义的等于和不等于，会使用is进行比较。

class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day
    def __eq__(self, other):
        return (self.year == other.year and self.month == other.month 
                and self.day = other.day)
x = Date(2023, 2, 24)
y = Date(2023, 2, 24)
print(x == y) # True

z = Date(2023, 5, 14)
print(x != z) # True

注意：当没有定义不等于的方法时，使用不等于运算符Python会对__eq__等于方法进行取反，所以通常情况下定义__eq__方法足够处理等于和不等于的运算了。当然也可以定义不等于的魔法方法__ne__。

class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day
    def __eq__(self, other):
        print("__eq__")
        return (self.year == other.year and self.month == other.month 
                and self.day == other.day)
    
    def __ne__(self, other):
        print("__ne__")
        return (self.year != other.year and self.month != other.month 
                and self.day != other.day)
    
x = Date(2023, 2, 24)
y = Date(2023, 2, 24)
print(x == y) 
# __eq__
# True

z = Date(2023, 5, 14)
print(x != z) 
# __ne__
# True

大于 __ gt__ 和小于__ lt__

__gt__：描述两个object进行比较时的过程，判断一个object是否大于另一个object。

class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day
    def __gt__(self, other):
        if self.year > other.year:
            return True
           if self.year == other.year:
            if self.month > other.month:
                return True
            if self.month == other.month:
                return self.day > other.day
            
x = Date(2023, 2, 22)
y = Date(2023, 2, 24)
print(x > y) # False
print(x < y) # True

当调用>大于运算符时，即 x > y 可以看作是x.__gt__(y)的运算。
注意：当没有实现小于运算却调用<小于运算符时，也可以正常返回结果，是因为x < y就对应y > x，所以可以认为进行了y.__gt__(x)的运算。

__lt__：描述两个object进行比较时的过程，判断一个object是否小于另一个object。当一个类里同时实现了__gt__和__lt__时，使用<运算符会优先调用__lt__。

class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day
    def __gt__(self, other):
        print("__gt__")
        if self.year > other.year:
            return True
           if self.year == other.year:
            if self.month > other.month:
                return True
            if self.month == other.month:
                return self.day > other.day
    def __lt__(self, other):
        print("__lt__")
        if self.year < other.year:
            return True
           if self.year == other.year:
            if self.month < other.month:
                return True
            if self.month == other.month:
                return self.day < other.day  
x = Date(2023, 2, 22)
y = Date(2023, 2, 24)
print(x < y) 
# __lt__
# True

因为x < y就对应y > x，x < y既可以通过x.__lt__(y)，也可以通过y.__gt__(x)，那么使用>或<到底调用哪个魔法方法呢？

调用原则：

  • 当x和y是同一个类的不同object，则优先使用运算符左侧的比较方法，当左侧object没有定义时，才会去使用右侧object的比较方法（不限于大于小于，还有等于不等于）。
  • 如果y是x的类的子类object，则优先使用y的比较方法（不限于大于小于，还有等于不等于）。
class NewDate(Date):
    pass

x = Date(2023, 2, 22)
y = NewDate(2023, 2, 24)
print(x < y)
# __gt__
# True

大于等于 __ ge__ 和小于等于__ le__

__ge__：与大于运算__gt__方法类似，greater than or equal to

__le__：与小于运算__lt__方法类似，less than or equal to

__ hash__

__hash__：求某个数据结构（自定义）的哈希值，一个自定义的数据结构，是有其默认的hash算法的，使用hash()可以直接拿到其哈希值。

class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day
        
x = Date(2023, 2,22)
print(hash(x)) # 8771107107837

如果要自定义hash算法，则一般也要定义__eq__方法，定义__hash__的要求如下：

  1. 返回值必须是整数
  2. 对于两个相等的object，其哈希值必须相同
class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day
    def __eq__(self, other):
        return (self.year == other.year and self.month == other.month 
        and self.day == other.day)
    def __hash(self):
        return 2

__ bool___

__bool__：描述object在条件判断语句中的行为。所有的object出现在条件判断语句中时，都会被判定为True，如果想要改变条件判断的结果，可以使用该方法。

class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day
a = Date(2023,2,23)
if a:
    print("hello")
# hello

class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day
    def __bool__(self):
        print("__bool__")
        return False
a = Date(2023,2,23)
print(bool(a)) # 当把object转变为bool时也会调用该方法
if a:
    print("hello")
# __bool__
# False
# __bool__

对象属性相关的方法

__ getattr__ 和 __ getattribute__

__getattr__：尝试访问object的属性时，描述该属性不存在时object的行为。默认情况下如果访问一个object里不存在的属性时，会抛出AttributeError。

class A:
    def __init__(self):
        self.exist = "abc"
    def __getattr__(self, name):
        print(f"getting {name}")
o = A()
print(o.exist) # abc
print(o.test) # getting test

注意：访问的该属性，会以字符串的形式传入到__getattr__里。只有在读取一个不存在的属性时才会调用该魔法方法。

__getattribute__：尝试访问object的属性时，不论该属性是否存在，都会调用该方法。

class A:
    def __init__(self):
        self.exist = "abc"
    def __getattribute__(self, name):
        print(f"getting {name}")
        return None
o = A()
print(o.exist) 
# getting exist
# None
print(o.test) 
# getting test
# None

__ setattr__ 和 __ delattr__

__setattr__：尝试写入object的属性时，描述该过程中object的行为。该方法有两个参数：一是属性名，二是属性值。一般情况下调用super().__setattr__()执行该方法的默认行为。

class A:
    def __init__(self):
        self.data = "abc"
        self.counter = 0
    def __setatter__(self, name, val):
        print(f"set{name}")
        super().__setattr__(name, val)
o = A()
print(o.data)
# set data
# set counter
# abc

__delattr__：与__del__不同，在一个object正常产生和消亡的过程中，不会调用__delattr__方法，而在尝试删除一个object的属性时才会调用。

class A:
    def __init__(self):
        self.data = "abc"
    def __delatter__(self, name):
        print(f"del{name}")
        super().__delattr(name)
        
o = A()
del o.data # del data

__ dir__

__dir__：列出object的所有属性名和方法名，Python规定该方法必须返回一个sequence（如列表list）。通过内置函数dir()调用该魔法方法。

>>>dir()   #  获得当前模块的属性列表
['__builtins__', '__doc__', '__name__', '__package__', 'arr', 'myslice']

a = list()
print(dir(a))    # 查看列表的方法
['__add__', '__class__', '__contains__', '__delattr__', '__delitem__', '__delslice__', '__doc__', '__eq__', '__format__', '__ge__', '__getattribute__', '__getitem__', '__getslice__', '__gt__', '__hash__', '__iadd__', '__imul__', '__init__', '__iter__', '__le__', '__len__', '__lt__', '__mul__', '__ne__', '__new__', '__reduce__', '__reduce_ex__', '__repr__', '__reversed__', '__rmul__', '__setattr__', '__setitem__', '__setslice__', '__sizeof__', '__str__', '__subclasshook__', 'append', 'count', 'extend', 'index', 'insert', 'pop', 'remove', 'reverse', 'sort']

class A:
    def __init__(self):
        self.data = "abc"
    def __dir__(self, name):
        return []
    
o = A()
print(dir(o)) # []

描述器

描述器的定义：在Python代码层面，如果一个类的内部定义了__get__()、__set__()或__delete__()这三个方法中的其中一个，则这个类被称为描述器。

如下代码是描述器最简单的用法，其中类Name就是一个描述器，内部定义了一个__get__()方法，该方的两个参数obj和objtype是必须的。

class Name:
    def __get__(self, obj, objtype):
        return "Peter"
    
class A:
    name = Name()

o = A()
print(o.name)
# Peter
print(A.name)
# Peter

从上面代码的运行结果可以推断出，调用o.name实际上是执行类Name的__get__方法，同样调用A.name也是执行类Name的__get__方法。这里的执行原理和Python内部关于描述器的机制有关，暂且不展开了。

__ get __

上述的代码中，描述器方法的self参数是描述器对象本身，也就是name，obj是对应类A的对象，也就是o，objtype是对象o的类型。

class Name:
    def __get__(self, obj, objtype):
        print(obj, objtype)
        return "Peter"
    
class A:
    name = Name()

o = A()
print(o.name)
# <__main__.A object at 0x7f87ab86a400> <class '__main__.A'>
# Peter

__ set __

通过__get__和__set__可以实现一个设置属性和读取属性的功能，与上述__setattr_和__getattr__相同的功能。不过描述器是class级别的(见下面代码的o2)。

class D:
    def __init__(self):
        self.val = 0
    def __get__(self, obj, objtype):
        return self.val
    def __set__(self, obj, value):
        self.val = value
class A:
    x = D()
    
o = A()
o2 = A()
print(o.x) # 0
o.x = 1
print(o.x) # 1
print(o2.x) # 1

__ delete __

当调用del o.x时会执行__delete__方法

class D:
    def __init__(self):
        self.val = 0
    def __get__(self, obj, objtype):
        return self.val
    def __set__(self, obj, value):
        self.val = value
    def __delete__(self, obj):
        print("delete")
class A:
    x = D()
    
o = A()
del o.x # delete

其他

__ slots __

__slots__不是一个方法，但是是一个有特殊含义的变量。它规定了当前这个类的对象可以有哪些自定义的属性，是一个类的白名单。

class A:
    __slots__ = ["a", "b"]

o = A()
o.x = 1 # 报错
o.a = 1 # 正常

迭代器：__ iter__ 和__ next__

迭代器就是重复地做一些事情，可以简单的理解为循环。

  • 迭代是访问集合元素的一种方式。
  • 迭代器是一个可以记住遍历的位置的对象。
  • 迭代器对象从集合的第一个元素开始访问，直到所有的元素被访问完结束。迭代器只能往前不会后退。
  • 迭代器有两个基本的方法：iter() 和 next()。

Python中实现了__iter__方法的对象是可迭代的，实现了__next__方法的对象是迭代器。实际上要想让一个迭代器工作，至少要实现__iter__方法和__next__方法。
常见的就是我们在使用for语句的时候，python内部其实在for后面的对象上使用了内建函数iter()：

a = [1, 2, 3]
for i in a:
    do_something()
    
# 其实内部做了如下转换
for i in iter(a):
    do_something()

下面引入两个概念：

  1. Iterable: 有迭代能力的对象。一个类实现了__iter__，那么就认为它有迭代能力，通常此函数必须返回一个实现了__next__方法的对象，如果自己实现了，则可以返回self，当然这个返回值不是必须的；
  2. Iterator: 迭代器(当然也是Iterable)。同时实现了__iter__和__next__的对象，缺少任何一个都不算是Iterator，比如下面例子中，A()可以是一个Iterable，但是A()和B()都不能算是和Iterator，因为A只实现了__iter__，而B只实现了__next__()。
class B(object):
    def __next__(self):
        raise StopIteration

class A(object):
    def __iter__(self):
        return B()

from collections.abc import *

a = A()
b = B()
print(isinstance(a, Iterable)) # True
print(isinstance(a, Iterator)) # False
print(isinstance(b, Iterable)) # False
print(isinstance(b, Iterator)) # False

创建一个迭代器

把一个类作为一个迭代器使用需要在类中实现两个方法**iter与next。
__iter__方法返回一个迭代对象， 这个迭代对象实现了__next__方法并通过StopIteration异常**来标识迭代的完成。
__next__方法会返回下一个迭代器对象。就可以通过next()函数访问这个对象的下一个元素了，并且在你不想继续有迭代的情况下抛出一个StopIteration的异常（for语句会捕获这个异常，并且自动结束for）

class MyRange(object):
    def __init__(self, end):
        self.start = 0
        self.end = end

    def __iter__(self):
        return self

    def __next__(self):
        if self.start < self.end:
            ret = self.start
            self.start += 1
            return ret
        else:
            raise StopIteration

a = MyRange(5)
for i in a:
    print(i) # 0 1 2 3 4

总结

  • 可以使用collection.abs里面的Iterator和Iterable配合isinstance函数来判断一个对象是否是可迭代的，是否是迭代器对象
  • iter()实际是映射到了__iter__函数
  • 只要实现了__iter__的对象就是可迭代对象(Iterable)，正常情况下应该返回一个实现了__next__的对象（强制要求，否则报错），如果自己实现了__next__，也可以返回自己。
  • 同时实现了__iter__和__next__的是迭代器(Iterator)，当然也是一个可迭代对象了，其中__next__应该在迭代完成后，抛出一个StopIteration异常
  • for语句会自动处理这个StopIteration异常以便结束for循环

生成器(yield表达式)

str，list，tuple，dict，set这些都是可迭代的，就是可用for来访问里面的每一个元素。但他们并不是迭代器。
生成器是一个可以快速创建迭代器的工具。生成器是可迭代的，更准确的说法是它就是个迭代器。
我们可以用列表生成式来初始化一个列表：

list1 = [x for x in range(5)]
print(list1) # [0,1,2,3,4]

gen = (x for x in range(5)) # 生成器表达式
print(gen) # generator object <genexpr> at 0x000000ADEF
# 调用gen的方法和迭代器一模一样
# 方法一
for x in gen:
    print(x)
# 方法二
print(next(gen))

在 Python 中，使用了yield的函数被称为生成器（generator）。生成器是一个返回迭代器的函数，只能用于迭代操作，更简单点理解生成器就是一个迭代器。

def MyRange(end):
    start = 0
    while start < end:
        yield start
        start += 1

from collections.abc import *

a = MyRange(2)
print(isinstance(a, Iterator)) # True
print(isinstance(a, Iterable)) # True
for i in a:
    print(i) # 0 1

可以看见，函数MyRange里面使用了yield语句，结果变成了generator，也是Iterator，我们在使用for语句的时候，执行步骤如下：

  1. start = 0， while 0 < 2，遇到了yield语句，返回start的值0，然后print这值；
  2. 第二次for的时候，执行yield后面的语句，start = 1，while 1<2，遇到了yield语句，返回start的值1，然后print；
  3. 第三次for的时候，执行yield后面的语句，start = 2，while 2<2不成立，此时函数结束运行，会抛出StopIterator的异常(这个是自动抛出的，不需要显示的调用raise语句，可以使用下面的next方法来查看这个异常)。

在调用生成器运行的过程中，每次遇到 yield 时函数会暂停并保存当前所有的运行信息，返回 yield 的值, 并在下一次执行 next() 方法时从当前位置继续运行。
