Python的多线程和多进程

进程和线程

  • 进程

进程是程序在计算机上的一次执行活动，即正在运行中的应用程序，通常称为进程。当你运行一个程序，你就启动了一个进程。每个进程都有自己独立的地址空间(内存空间)，每当用户启动一个进程时，操作系统就会为该进程分配一个独立的内存空间，让应用程序在这个独立的内存空间中运行。

在同一个时间里，同一个计算机系统中如果允许两个或两个以上的进程处于运行状态，这便是多进程，也称多任务。现代的操作系统几乎都是多任务操作系统，能够同时管理多个进程的运行。多任务带来的好处是明显的，比如你可以边听mp3边上网，与此同时甚至可以将下载的文档打印出来，而这些任务之间丝毫不会相互干扰。

  • 线程

线程是一个轻量级的子进程，是最小的处理单元，是一个单独的执行路径。可以说：线程是进程的子集（部分），一个进程可能由多个线程组成。线程是相互独立的，如果在一个线程中发生异常，则不会影响其他线程。同一个进程内的线程之间使用共享内存区域。

多线程是一种执行模型，它允许多个线程存在于进程的上下文中，以便它们独立执行但共享其进程资源。

  • 线程和进程的区别

1）一个进程至少包含一个主线程（main线程），线程需要依赖于进程而存在。
2）多个线程之间共享进程的内存空间。

调度：进程是拥有资源的基本单位，是系统资源分配的最小单位；线程是调度和分派的基本单位，是CPU资源调度的最小单位。

共享地址空间：进程拥有各自独立的地址空间、资源，所以共享复杂，需要用IPC（Inter-Process Communication，进程间通信），但是同步简单；而线程共享所属进程的资源，因此共享简单，但是同步复杂，需要用加锁等措施。
占用内存和cpu：进程占用内存多，切换复杂，cpu利用率低；而线程占用内存少，切换简单，cpu利用率高。
互相影响：进程之间不会互相影响；而一个线程挂掉会导致整个进程挂掉。

多线程threading

多线程是加速程序计算的有效方式，Python的多线程模块threading上手快速简单，包含了关于线程操作的丰富功能。

创建线程

threading模块的一些基本操作，包括获取线程数，添加线程等。

import threading
print(threading.active_count()) # 获取已激活的线程数
print(threading.enumerate()) # 查看所有线程信息---存储在一个列表中
print(threading.current_thread()) # 查看现在正在运行的线程

添加线程，threading.Thread()接收参数target代表这个线程要完成的任务，需自行定义，start()让线程开始工作。

def thread_job():
    print('This is a thread of %s' % threading.current_thread())

def main():
    thread = threading.Thread(target=thread_job,)   # 定义线程 
    thread.start()  # 让线程开始工作
    
if __name__ == '__main__':
    main()

在定义线程时，还可以指定线程的名称，如下面的代码：

thread = threading.Thread(target=thread_job,)   # 定义线程, 默认线程名是Thread-1
thread = threading.Thread(target=thread_job, name="T1")   # 定义线程, 指定名称是T1

还可以通过自定义一个 Thread 的子类，然后复写它的 run() 方法，在 run() 方法中编写任务处理代码，实现自定义的多线程程序。

import threading
import time

class TestThread(threading.Thread):
    def __init__(self,name=None):
        super().__init__(self,name=name)

    def run(self):
        for i in range(5):
            print(threading.current_thread().name + ' test ', i)
            time.sleep(1)
            
thread = TestThread(name='TestThread')
thread.start()

join功能

首先来看一个小例子，下面定义了一个任务函数func，执行for循环10次，每次睡眠1秒，执行完成之后打印"all done"。但是我们从结果看到，打印信息"all done"先于"thread finish"被输出。

import time
import threading

def func():
    print("thread start")
    for i in range(10):
        time.sleep(1)
    print("thread finish")

t1 = threading.Thread(target=func, name="t1")
t1.start()
print("all done")

>>> Output:
thread start
all done
thread finish

这是因为这里有两个线程，分别是主线程MainThread和定义的线程t1，这两个线程同时运行时，由于t1需要执行函数func，共计需要10s才能完成，而主线程运行不需要这么久，因此主线程会先运行完毕输出"all done"，等t1执行函数func完成时才会输出"thread finish"。

因此，如果想让所有线程执行完毕后，再一起执行后面的代码（比如上面的输出all done），可以使用join方法。如下面的代码

def func(num):
    print("%s thread start"%(threading.current_thread()))
    for i in range(num):
        time.sleep(1)
    print("%s thread finish"%(threading.current_thread()))

t1 = threading.Thread(target=func, args=(10,), name="t1")
t2 = threading.Thread(target=func, args=(1,), name="t2")

上面添加了两个线程t1和t2，均执行函数func，但是两个线程执行任务的耗时不同，t1执行任务10s，而t2执行任务1s。下面分别使用join方法来观察输出。

t1.start()
t2.start()
print("all done")

Queue功能

线程间使用队列进行通信，因为队列所有方法都是线程安全的，所以不会出现线程竞争资源的情况。queue 模块即队列，特别适合处理信息在多个线程间的安全交换。

多线程调用的函数不能有返回值，所以最直白的应用，就是多个线程处理完毕的结果，放入该队列中，最后进行整合。

import time
import threading
from queue import Queue

def square(d, q):
    for i in range(len(d)):
        d[i] = d[i]**2
    q.put(d)

def multithreading(data):
    q = Queue()
    threads = list()
    for i in range(4):
        t = threading.Thread(target=square, args=(data[i], q))
        t.start()
        threads.append(t)
    for thread in threads:
        thread.join()
    results = list()
    for _ in range(4):
        results.append(q.get())
    print(results)

data = [[1,2,3],[3,4,5],[2,2,2],[3,3,3]]
multithreading(data)

其中q.put()和q.get()均是阻塞式的。q.put()阻塞方式将数据添加进队列中，如果队列满了则一直等待；q.get()阻塞式获取，队列为空时，则一直等待。

这里的队列也可以使用multiprocessing提供的Queue。

Lock锁

多个线程可以共享所属进程的资源，但是如果不加限制，则线程之间会对共享的资源进行争抢，导致无法预估的问题。比如下面的代码，两个线程共享一个变量A，如果不对共享的内存A进行限制，则最终的结果A是未知的。

import threading

def job1():
    global A
    for i in range(10):
        A += 1
        print('job1', A)
def job2():
    global A
    for i in range(10):
        A += 10
        print('job2', A)

A = 0
t1 = threading.Thread(target=job1)
t2 = threading.Thread(target=job2)
t1.start()
t2.start()
t1.join()

因此，需要使用锁（Lock）来限制多个线程对同一块内存区域的访问。

def job1():
    global A, lock
    lock.acquire() # 加锁
    for i in range(10):
        A += 1
        print('job1', A)
    lock.release() # 释放
def job2():
    global A,lock
    lock.acquire()
    for i in range(10):
        A += 10
        print('job2', A)
    lock.release()

A = 0
lock = threading.Lock()

多线程不一定有效率

多线程的工作原理：同一时间只有一个线程在工作，多个线程来回切换，实现多个线程“同时工作”的假象。多线程往往处理并发，即收到多个请求后，多个线程每次处理一点，线程来回切换完成多个请求任务。

只有当多个线程处理的任务存在明显的效率差异时，如遇到IO，可以使用多线程提高速度和整体效率；如果是计算密集型任务，把数据拆分给多个线程运算，则不会带来很大的效率收益。

多进程multiprocess

多进程是利用多核CPU进行任务处理和运算的一种方式。Python中的multiprocessing模块提供了一种创建和管理进程的方式，使得可以利用多个CPU来加速程序运行。

创建进程

多进程的创建和多线程类似，代码如下：

import multiprocessing as mp

def job(a,b):
    print("hello")
p1 = mp.Process(target=job, args=(1,2))
p1.start()
p1.join()

同样地，也可以自定义一个多进程处理的继承自Process的子类，然后复写它的 run() 方法，在 run() 方法中编写任务处理代码，实现自定义的多进程程序。

from multiprocessing import Process

class NewProcess(Process):
    def __init__(self, name):
        super().__init__()  
        
        # 创建新参数
        self.name = name
    
    # 在自定义Process类时，必须实现run()方法
    def run(self):
        print(f'{self.name}: Hello World')

Queue功能

与多线程类似，多线程调用的函数不能有返回值，因此可以把多个进程的处理或运算结果，放入到队列中，等所有进程均执行完毕后进行整合。

import multiprocessing as mp

def job(q):
    res = 0
    for i in range(1000):
        res += i+i**2+i**3
    q.put(res)
q = mp.Queue()
p1 = mp.Process(target=job, args=(q,))
p2 = mp.Process(target=job, args=(q,))
p1.start()
p2.start()
p1.join()
p2.join()
res1 = q.get()
res2 = q.get()
print(res1+res2)

多线程和多进程的效率对比

import multiprocessing as mp
import threading as td
import time

def job(q):
    res = 0
    for i in range(100000):
        res += i+i**2+i**3
    q.put(res)

def normal():
    res = 0
    for _ in range(2): # 对齐两个进程或线程
        for i in range(100000):
            res += i+i**2+i**3
    print("normal", res)
def multicore():
    q = mp.Queue()
    p1 = mp.Process(target=job, args=(q,))
    p2 = mp.Process(target=job, args=(q,))
    p1.start()
    p2.start()
    p1.join()
    p2.join()
    res1 = q.get()
    res2 = q.get()
    print("multicore", res1+res2)
def multithread():
    q = mp.Queue()
    t1 = td.Thread(target=job, args=(q,))
    t2 = td.Thread(target=job, args=(q,))
    t1.start()
    t2.start()
    t1.join()
    t2.join()
    res1 = q.get()
    res2 = q.get()
    print("multithread", res1+res2) 

st = time.time()
normal()
st1 = time.time()
print("normal time ", st1-st)
multithread()
st2 = time.time()
print("thread time ", st2-st1)
multicore()
print("core time ", time.time()-st2)

进程池Pool

与Process创将进程不同，这次使用Pool创建进程池，它会自行解决多进程的问题。Pool和之前的Process的不同点是传入Pool的函数有返回值，而Process传入的函数没有返回值。

import multiprocessing as mp

def job(x):
    return x*x 
    
def multicore():
    pool = mp.Pool()
    res = pool.map(job, range(10))
    print(res) # [0,1,4,9,16,25,36,49,64,81]

map接受数据自动分配给进程池的进程进行运算。Pool默认使用所有的CPU核，如果想只用其中几个核，则由参数process指定。

pool = mp.Pool(processes=3) # 使用3个核，即3个进程

Pool除了map()外，还有可以返回结果的方式，那就是apply_async()。apply_async()中只能传递一个值，它一次只会放入一个核进行运算，但是传入值时要注意是可迭代的，同时需要用get()方法获取返回值。

def multicore():
    pool = mp.Pool(processes=3) 
    res = pool.apply_async(job, (2,))
    # 用get获得结果
    print(res.get()) # 4

如果想要传入多个数值，则需要使用迭代器。

def multicore():
    pool = mp.Pool(processes=3) 
    multi_res = [pool.apply_async(job, (i,)) for i in range(10)]
    # 用get获得结果
    print([res.get() for res in multi_res]) # 4

共享内存

进程拥有各自独立的地址空间、资源，因此想要在进程之间共享是没有多线程方便的（global关键字）。共享内存是一种高效的进程间通信方式，它允许多个进程共享同一块内存区域。multiprocessing模块中提供了Value和Array类，可以用来创建共享内存。

value1 = mp.Value('i', 0) 
value2 = mp.Value('d', 3.14)

array = mp.Array('i', [1, 2, 3, 4])

这里的Array和numpy中的不同，它只能是一维的，不能是多维的。Value和Array 一样，需要定义数据形式，否则会报错。d表示一个双精浮点类型，i表示一个带符号的整型。具体数据类型见下表。

'H' unsigned short int 2

Lock锁

上面介绍了共享内存的概念，如果多进程程序不对共享内存加以限制，则会发生多个进程对共享内存的争夺。因此锁出现了，它是为了防止多个进程对共享内存进行读写，导致无法预估的事情发生。

import multiprocessing as mp
import time
 
def job(v, num):
    for _ in range(5):
        time.sleep(0.1) 
        v.value += num # v.value获取共享变量值
        print(v.value, end=" ")
        
def multicore():
    v = mp.Value('i', 0) # 定义共享变量
    p1 = mp.Process(target=job, args=(v,1))
    p2 = mp.Process(target=job, args=(v,10)) 
    p1.start()
    p2.start()
    p1.join()
    p2.join()
    
multicore() # 1 12 23 34 45 11 22 33 44 55

与多线程类似，加入Lock限制共享内存。

import multiprocessing as mp
import time
 
def job(v, num, lock):
    lock.acquire()
    for _ in range(5):
        time.sleep(0.1) 
        v.value += num # v.value获取共享变量值
        print(v.value, end=" ")
    lock.release()
        
def multicore():
    lock = mp.Lock()
    v = mp.Value('i', 0) # 定义共享变量
    p1 = mp.Process(target=job, args=(v,1, lock))
    p2 = mp.Process(target=job, args=(v,10, lock)) 
    p1.start()
    p2.start()
    p1.join()
    p2.join()
    
multicore() # 1 2 3 4 5 15 25 35 45 55
